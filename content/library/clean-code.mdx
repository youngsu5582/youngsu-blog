## 핵심 내용

깨끗한 코드란 무엇인가에 대한 고민을 담은 책입니다.

### 인상 깊었던 부분

> "코드는 요구사항을 상세히 표현하는 언어다"

함수는 한 가지만 해야 하고, 이름은 의도를 드러내야 합니다.

## 주요 원칙

### 1. 의미 있는 이름

```java
// Bad
int d; // elapsed time in days

// Good
int elapsedTimeInDays;
int daysSinceCreation;
```

### 2. 함수

- **작게 만들어라** - 함수는 한 가지 일만 해야 한다
- **서술적인 이름을 사용하라** - 함수가 하는 일을 더 잘 표현하는 이름
- **인수는 적을수록 좋다** - 0개가 가장 좋고, 3개 이상은 피하라

### 3. 주석

좋은 코드는 주석이 필요 없다. 주석은 코드의 실패를 의미한다.

```java
// Bad - 주석으로 의도를 설명
// Check to see if the employee is eligible for full benefits
if ((employee.flags & HOURLY_FLAG) && (employee.age > 65))

// Good - 함수 이름으로 의도를 표현
if (employee.isEligibleForFullBenefits())
```

## 느낀 점

코드의 품질은 시간이 지나도 유지보수할 수 있는지로 판단해야 합니다.
당장 동작하는 코드보다 읽기 쉽고 변경하기 쉬운 코드가 좋은 코드입니다.

프로페셔널 개발자라면 항상 코드를 깨끗하게 유지하려는 노력이 필요합니다.
