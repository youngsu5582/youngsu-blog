## 기술 스택 결정 과정

블로그를 처음부터 만들기로 결정한 후, 기술 스택을 선택하는 과정을 기록합니다.

### Framework 선택

| 후보 | 장점 | 단점 |
|------|------|------|
| Next.js | 최대 생태계, SSG/SSR | 번들 크기 |
| Astro | 빠른 빌드, 최소 JS | 동적 기능 제한 |
| Hugo | 빌드 속도 최강 | Go 템플릿 러닝커브 |

### 최종 선택: Next.js 16

```bash
pnpm create next-app@latest youngsu-blog --typescript --tailwind --app
```

React 생태계의 풍부한 자원과 Vercel의 완벽한 지원이 결정적이었습니다.

### Tailwind CSS v4.2

제로 설정으로 바로 사용 가능하고, 빌드 속도가 5배 빨라졌습니다.

> "CSS를 작성하지 않고도 아름다운 UI를 만들 수 있다"

### shadcn/ui

컴포넌트를 직접 소유하므로 완전한 커스터마이징이 가능합니다.
