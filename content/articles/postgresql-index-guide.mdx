## B-Tree 인덱스

PostgreSQL의 기본 인덱스 타입입니다.

```sql
CREATE INDEX idx_users_email ON users (email);
```

### 언제 사용하나?

- 등호(=), 범위(`<`, `>`, BETWEEN) 비교
- ORDER BY 최적화
- 유니크 제약조건

## Hash 인덱스

등호 비교에만 사용되는 인덱스입니다.

```sql
CREATE INDEX idx_users_id ON users USING HASH (id);
```

### 특징

- PostgreSQL 10부터 WAL 지원으로 안정성 확보
- 등호 검색에서는 B-Tree보다 빠를 수 있음
- 범위 검색 불가

## GiST 인덱스

기하학적 데이터나 전문 검색에 사용됩니다.

```sql
CREATE INDEX idx_locations ON places USING GIST (location);
```

## 인덱스 최적화 전략

1. **선택도가 높은 컬럼 우선** - 카디널리티가 높은 컬럼에 인덱스 생성
2. **복합 인덱스 순서** - 자주 사용되는 컬럼을 앞에 배치
3. **인덱스 크기 모니터링** - `pg_indexes` 뷰로 확인

```sql
SELECT
  schemaname,
  tablename,
  indexname,
  pg_size_pretty(pg_relation_size(indexrelid)) AS size
FROM pg_indexes
JOIN pg_class ON indexrelid = pg_class.oid
ORDER BY pg_relation_size(indexrelid) DESC;
```
